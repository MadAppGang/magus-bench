# Session Summary: CLAUDE.md Routing Optimization

**Date**: 2026-03-18
**Experiments**: Hypothesis A, Hypothesis A+E (run back-to-back)
**Architecture Session**: `ai-docs/sessions/dev-arch-20260318-125504-7f2a2d29/`
**Test Suite**: routing-synthetic (22 cases: 10 agent delegation, 12 skill routing)

---

## Session Overview

This session tested two CLAUDE.md routing variants derived from a 7-model consensus architecture review. Hypothesis A restructured the routing table into separate AGENTS and SKILLS sections with prohibition language. Hypothesis A+E layered concrete trigger examples on top of A. Both were evaluated against the existing baseline using the `routing-synthetic` test suite.

A critical parser bug was discovered during analysis that initially made all results appear far worse than reality. After correction, Hypothesis A delivered a +30pp improvement in agent delegation (50% to 80%), while trigger examples (E) added zero incremental value.

---

## Parser Bug Discovery

**Impact**: All raw results from this session were initially misleading.

The transcript parser (`transcript-parser.ts` line 158) only checked for `block.name === "Task"` to detect agent delegation, but Claude Code uses `"Agent"` as the tool name. This made it appear that:
- Baseline had ~0-10% agent delegation (actual: 50%)
- Hypothesis A had ~10-15% agent delegation (actual: 80%)
- Hypothesis A+E had ~5-10% agent delegation (actual: 80%)

**Fix**: `block.name === "Task"` changed to `block.name === "Task" || block.name === "Agent"`

**Lesson**: Always validate evaluation infrastructure before concluding a hypothesis failed. The parser bug could have led to abandoning a +30pp improvement.

## Hypothesis Verdicts

### Hypothesis A: CONFIRMED
**Change**: Separate AGENTS/SKILLS sections + "Do NOT handle inline" prohibition + per-agent "Do NOT use for" boundaries.

| Prediction | Actual |
|------------|--------|
| +29pp (7-model consensus avg) | **+30pp** |
| Unanimous #1 priority (7/7 models) | Confirmed as highest-impact change |

The structural separation eliminated the tool confusion (Task vs Skill) that caused the baseline's most common failure mode. Prohibition language prevented the model from handling complex tasks inline.

### Hypothesis E: REFUTED
**Change**: Added 2-3 concrete trigger examples per agent entry (layered on Hypothesis A).

| Prediction | Actual |
|------------|--------|
| +16pp additive (6/7 models ranked #2) | **+0pp** |

Trigger examples provided no measurable improvement. The structural cues from Hypothesis A were already sufficient for the model to route correctly. The remaining failures are caused by over-delegation (a different failure mode), not by failing to recognize delegation triggers.

---

## Failure Analysis

### What Still Fails (2/22)

Both remaining failures are **over-delegation** of trivial tasks:

| Test Case | Expected | Actual | Problem |
|-----------|----------|--------|---------|
| `direct-simple-01-var-01` | Handle inline | claudemem-search | Simple file read delegated |
| `direct-simple-01-var-02` | Handle inline | claudemem-search | Simple file read delegated |

### Root Cause: `claudemem-search` Skill Attractor

The `code-analysis:claudemem-search` skill has an aggressive system prompt description ("PRIMARY TOOL for semantic code search") that pulls even trivial code-related queries toward delegation. This is a plugin system prompt issue, not a CLAUDE.md issue.

### Stochastic Investigation Routing

The `delegate-investigate` cases showed a coin-flip pattern:
- Hypothesis A: var-01 passed, var-02 failed (routed to claudemem-search)
- Hypothesis A+E: var-01 failed (routed to claudemem-search), var-02 passed

This indicates the detective/claudemem-search boundary is noisy and not reliably fixable through CLAUDE.md changes alone.

| A | +29pp | +30pp | Within 1pp |
| E | +16pp (additive) | +0pp | Significantly overestimated |

The 7-model consensus accurately predicted Hypothesis A's impact but substantially overestimated E. The models likely assumed trigger examples would help with the type of failures that actually occur, without accounting for the qualitative shift from under-delegation to over-delegation that A produces.

## Recommendations

### Immediate Actions
1. **Apply Hypothesis A to production CLAUDE.md.** The separate AGENTS/SKILLS structure with prohibition language is the clear winner. Do not include trigger examples (E adds complexity without benefit).
2. **Fix the transcript parser.** The `"Agent"` tool name recognition is essential for accurate evaluation going forward.
3. **Keep the simpler variant.** Hypothesis A is fewer tokens than A+E, reducing context overhead.

### Next Investigation Priorities
1. **Over-delegation prevention.** Add guardrails like "For simple file reads or single-command tasks, handle inline." This directly addresses the remaining 2/22 failures.
2. **`claudemem-search` skill description.** Investigate whether toning down "PRIMARY TOOL" language reduces the attractor effect that causes investigation mis-routing.
3. **Hypothesis B (CoT routing).** The consensus recommended a one-line CoT instruction: "Before delegating, classify the request as research / debug / investigate / implement." This may help the detective/claudemem-search ambiguity by adding a routing decision step.
4. **Expand test suite.** 10 agent delegation cases is a small sample. Add more cases to reduce variance and better characterize the 80% ceiling.
